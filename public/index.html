How it works
Up and running in three steps

One webhook. No SDKs. No servers to manage. CronVend handles the scheduling, sequencing, and sending.

Step 01
🔗

Connect your store

Point your order management system at your CronVend webhook endpoint. Send order events with a single HTTP POST — no SDK required.

POST /api/webhook/orders
x-api-key: cv_live_…

{ "event": "order.pending",
  "orderId": "ORD-7821",
  "email": "jane@shop.co" }
Step 02
⏱️

Sequences run on schedule

CronVend's cron engine wakes every hour. For each pending order, it automatically sends payment reminders, follow-ups, and promo codes on a tested cadence.

Step 03
💰

Watch orders recover

When a customer completes payment, CronVend detects it and stops the sequence instantly — no extra emails, no awkward double-sends.

Features
Everything included, nothing to configure
📅

Timed email sequences

Payment reminders at 1 h, 24 h, and 72 h. Review follow-ups after delivery. Promo offers for stubborn holdouts. Tuned timing, out of the box.

🎟️

Automatic promo codes

CronVend generates unique discount codes per order and embeds them in emails. Redeemed codes are tracked so you know exactly what converted.

🛑

Instant opt-out compliance

Every email includes a one-click unsubscribe. Opt-outs are stored immediately — that customer never receives another email from your sequence.

📊

Delivery event tracking

Sent, delivered, opened, clicked, bounced, complained — every event is recorded per order and visible in your dashboard in real time.

🏢

Multi-tenant by design

Built for agencies and resellers. Each client gets a fully isolated workspace, their own API key, and their own sending quota — completely separate.

🔒

GDPR data erasure

A single API call deletes all stored data for a tenant — orders, events, rate limits, API keys — in full compliance with right-to-erasure requests.
